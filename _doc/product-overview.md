# Product Overview — Horror Forest 100

## Working Title
Horror Forest 100 *(final name TBD — to be confirmed in onboarding)*

## What It Is
A browser-based horror survival game. The player is stranded deep inside a haunted forest, sheltered in a lone house, and must survive **100 consecutive nights** against escalating ghost encounters. The game is built around a single, unambiguous win condition — outlast all 100 nights — giving it a defined challenge arc that most browser horror games lack.

## Target Users
- Horror game enthusiasts who want a browser-accessible, pick-up-and-play scare
- Casual to mid-core gamers drawn to defined challenge-run formats
- Players motivated by shareable achievements ("I survived all 100 nights")

## Core Game Loop
1. **Day phase** — Scavenge the surrounding forest for supplies, repair structural damage, board up windows, set traps, manage power/light
2. **Night phase** — Ghost AI activates; player must repel or evade ghost encounters using barricades, light management, and rationed resources
3. **Escalation** — Each new night raises the threat level: more ghosts, smarter patterns, new ghost types
4. **Win/Lose** — Survive night 100 = victory screen with shareable result; caught by a ghost = game over with fast-retry to night 1 (or last checkpoint)

## Key Features
- **100-night challenge arc** — strict, defined, beatable win condition
- **Ghost AI escalation** — threat compounds across 100 nights; no two night ranges feel the same
- **House fortification system** — board windows, lock/barricade doors, manage electricity and light sources
- **Resource management** — day scavenging feeds night survival; scarcity drives tension
- **Atmospheric horror** — dark forest visuals, ambient sound design, fog, minimal UI for immersion
- **Score / streak tracking** — "nights survived" as the primary metric; personal bests and leaderboard
- **Fast retry loop** — game over to back in action in seconds; compulsion engine

## Brand & Tone
- **Tone**: Atmospheric horror — tense, slow-burn, genuinely unsettling; not gratuitously gory
- **Visual palette**: Near-black backgrounds, deep forest greens, blood-red and amber accents, moonlit fog
- **Feel**: Isolation and dread — the player should feel alone, outmatched, and compelled to try again

## Platform
Web browser (QuantumByte-built application)

## Positioning
The only browser horror game with a strict 100-night challenge format. Unlike one-shot jump-scare pages or bloated survival sims, this game has a real finish line — making it a horror run players can actually *beat* and brag about.

## Strategic Principles
1. **Atmosphere over complexity** — every mechanic exists to deepen the scare
2. **Clear win condition** — 100 nights drives motivation and social sharing
3. **Escalating difficulty** — keeps nights 1–100 progressively terrifying, no plateau
4. **Replayability via retry** — fast restart and high-score loop keep players returning
